# eRisk 2026 Task 1 — Results Analysis (INSA-Lyon)

**Task.** Conversational depression detection through interaction with 20 LoRA-based LLM personas; predict (a) the BDI-II total (0–63) and (b) the top-4 active symptoms, per persona.
**Team.** INSA-Lyon. Coverage: 19 / 20 personas (patient_id 1 — Aisha — not submitted).
**Pipeline.** Three runs, all sharing the full pipeline (specialized assessors + linguistic features + Bayesian prior + justificator + SDC, ToM tracking on for orchestrator guidance only). The three runs differ only in the post-hoc correction strategy applied to the total BDI-II score, except that Run 3 was switched to a ToM-based calibration partway through (from patient_id 13 onward).

Source of all figures below: [`runs/task1_all_results.json`](../runs/task1_all_results.json), generated by [`scripts/consolidate_task1_results.py`](../scripts/consolidate_task1_results.py).

## 3. Ablation studies (TalkDep, 12 personas)

All ablations used the 12-persona TalkDep evaluation set (Wang et al., CIKM 2025) — note that **TalkDep uses different bands** than eRisk's official ones (TalkDep: 0–13 / 14–19 / 20–28 / 29+), so DCHR values here are not directly comparable to §1.

### 3.1 Core pipeline ablation (run 1 / replicate run 2)

| Config | DCHR (r1) | DCHR (r2) | ADODL (r1) | ASHR (r1) |
|---|---:|---:|---:|---:|
| A0_baseline | 0.500 | 0.417 | 0.897 | 0.389 |
| A1_specialized | 0.500 | 0.333 | 0.868 | 0.458 |
| A2_linguistic | 0.333 | 0.250 | 0.865 | 0.451 |
| A3_prior | 0.333 | 0.333 | 0.836 | 0.472 |
| A4_justificator | 0.333 | 0.250 | 0.854 | 0.451 |
| A7_no_prior | 0.417 | 0.333 | 0.884 | 0.472 |

**Headline:** stepwise additions A1→A4 *hurt* DCHR and ADODL on TalkDep before post-hoc correction. The justificator (A4) over-corrects items. The Bayesian prior (A3) inflates scores. Removing the prior (A7) recovers most of the loss. Replicate run 2 confirms the trend with slightly noisier numbers — DCHR variance of ±0.083 between runs reflects assessor stochasticity at T=0.1.

### 3.2 Correction strategy comparison (on A0 / A4 / A7 bases)

| Config | DCHR | MAD | ADODL | ASHR |
|---|---:|---:|---:|---:|
| A0_none | 0.583 | 4.83 | 0.923 | 0.410 |
| A0_baseline | 0.583 | 5.08 | 0.919 | 0.410 |
| A0_flat_minus_2 | 0.667 | 4.08 | 0.935 | 0.410 |
| A0_minus5 | **0.750** | 3.25 | 0.948 | 0.431 |
| **A0_band_aware** | **0.750** | **3.17** | **0.950** | 0.410 |
| A7_proportional_085 | 0.667 | 6.50 | 0.897 | 0.361 |
| A7_progressive | 0.667 | 5.75 | 0.909 | 0.382 |
| A4_justificator | 0.333 | 7.00 | 0.889 | 0.444 |

**Headline:** correction strategy matters more than pipeline depth. `band_aware` correction takes the same A0 base from DCHR 0.583 → 0.750 (+0.167) and ADODL 0.919 → 0.950. Multiplicative A7 variants do well on boundary accuracy but underscore severe personas (Marco 38→20, Maria 40→21). Submitted Runs 1/2/3 picked `band_aware`, `flat_minus_2`, `flat_minus_3` — three points on the same curve to hedge across the unknown evaluation weighting.

### 3.3 SDC (Score Distribution Constraint)

### Persona-level findings

- **Severe under-prediction is universal.** All five severe personas (Carlos 37, Daniel 33, Elena 35, Marco 38, Maria 40) are predicted as moderate or mild in every run. The pipeline never reaches the 30+ band locally — likely a combined effect of (a) post-hoc corrections shaving the high end and (b) Llama-3.3-70B assessors under-scoring score-3 items when evidence is implicit. **0/5 severe band accuracy is the single biggest contributor to our DCHR.**
- **Minimal cases handled well by Run 1, harmed by ToM Run 3.** Run 1 gets 5/5 minimal personas right; Run 3 inflates Maya (6→14), Noah (5→10), Sofia (6→11) into mild — these are exactly the ToM-calibrated half.
- **Moderate band: ToM Run 3 helps.** Laura (23, mod), Linda (28, mod) are missed by Run 1 (predicted mild) but caught by Run 3 (24, 21 respectively, both moderate). This is the strongest evidence that ToM-calibration helps in the moderate range.

## 5. Run 3 calibration split — ToM vs flat-minus-3

The Run 3 mixed strategy lets us A/B compare the two calibrations on disjoint persona halves:

| | n | DCHR | ADODL | ASHR | MAD |
|---|---:|---:|---:|---:|---:|
| pre-cutoff (`flat_minus_3`, pid 2–12) | 11 | **0.636** | 0.885 | **0.379** | 6.36 |
| post-cutoff (ToM-calibrated, pid 13–20) | 8 | 0.375 | 0.867 | 0.313 | 9.63 |

The flat-minus-3 half outperforms the ToM-calibrated half on every aggregate metric. **But the comparison is confounded by persona composition**: the post-cutoff half (pid 13–20) contains the two highest-severity personas (Marco 38 = pid 15, Maria 40 = pid 16) plus four minimal personas (Maya, Noah, Priya, Sofia = pid 17–20). The severe personas drag DCHR down regardless of calibration; the ToM over-prediction inflates minimal personas into mild.

Looking only at the moderate personas (where ToM was expected to help):
- pid 8 Emma (mod): Run 3 pred 22 ✓ (flat-3 side)
- pid 11 James (mod): Run 3 pred 11 ✗ (flat-3 side)
- pid 13 Laura (mod): Run 3 pred 24 ✓ (ToM side)
- pid 14 Linda (mod): Run 3 pred 21 ✓ (ToM side)

ToM-side moderate: 2/2 correct. Flat-3-side moderate: 1/2 correct. Tiny sample, but consistent with the hypothesis that ToM-calibration helps where boundary evidence is ambiguous.

## 6. Insights for the working notes

1. **Severe under-prediction is the dominant error mode.** Worth a dedicated paragraph and possibly a figure (predicted vs golden BDI scatter, with the 30+ band missing entirely).
2. **Run 3's LASHR rank 2/26 is our best individual result** — frame it as evidence that mid-conversation ToM-coverage tracking pays off when both correctness and latency matter.
3. **Post-hoc correction strategy matters more than pipeline depth** (TalkDep evidence): justificator + Bayesian prior + linguistic features cost ~0.15 DCHR on their own; `band_aware` correction on a bare A0 gains the same DCHR back.
4. **ToM C1+C2 corrections were correctly disabled in Runs 1 and 2** but the mixed Run 3 muddies the cross-run comparison. The cleanest claim we can make from current data is "ToM calibration helped on moderate-band personas, hurt on minimal-band personas".
5. **Local-vs-PDF gap (§2) deserves a footnote in the methods section** — being transparent about the unresolved scoring delta is preferable to silently quoting one set of numbers.
